# 局域网屏幕墙 (LAN Screen Wall)

一个干净、开源、可审计的局域网屏幕墙工具。一台主控机以网格形式实时展示局域网内多台终端的屏幕画面，适用于机房、教室、办公区监控等场景。

## 为什么重写这个工具

原版"屏幕墙-局域远控工具1.3.exe"被杀毒软件报毒。经静态分析，报毒主要源于：
- 用 **易语言** 编写（易语言程序常被启发式杀毒误报）
- 使用了 **破解版 SkinSharp 换肤库**（`SkinH_EL.dll`，FileDescription 标注"破解版"）
- 含 `WriteProcessMemory` 等进程注入 API、内嵌多个子 exe（`server3.exe`/`up64.exe`/`setup.exe` 等），行为不透明

本重写版用 **Python** 从零实现，**不注入进程、不读写文件、不修改注册表、不内嵌子程序**，全部行为透明可审计，且仅做"屏幕采集 + 上墙展示"，不做远程控制（避免高危行为）。

## 架构

```
┌──────────────┐   TCP/长连接    ┌──────────────┐
│  采集端 agent │ ───────────────▶│ 主控端 hub    │
│ (每台被监控机)│  JPEG 帧流      │ (监控机, GUI) │
└──────────────┘                 │  网格展示     │
       ...多台...                 └──────────────┘
```

- **采集端 `agent.py`**：抓取本机屏幕 → JPEG 压缩 → 推送给主控端，断线自动重连
- **主控端 `hub.py`**：监听端口接收画面 → 网格自动排布 → 实时显示，支持放大/调帧率/调画质/断开
- **协议 `protocol.py`**：长度前缀帧 + 令牌握手，简单健壮

## 安全设计

| 项目 | 做法 |
|------|------|
| 认证 | 连接必须携带令牌(Token)，主控端校验后才接受画面，令牌不符直接拒绝 |
| 权限 | 采集端**只发送**本机画面，不接受任何远程指令（仅接受帧率/画质参数） |
| 行为 | 无进程注入、无文件读写、无注册表修改、无自启、无子进程释放 |
| 依赖 | 全部为知名开源库(PyQt5/mss/Pillow)，可审计 |
| 网络 | 仅局域网 TCP，无外联 |

## 安装

需要 Python 3.8+，在主控机和每台终端上执行：

```bash
pip install -r requirements.txt
```

## 使用

### 1. 启动主控端（监控机）

双击 `run_hub.bat`，或命令行：

```bash
python hub.py --port 5000 --token mySecret
```

参数：
- `--port` 监听端口（默认 5000）
- `--token` 连接令牌（默认 change-me，**务必修改**）
- `--cols` 固定列数，0=自动按数量排布（默认 0）

### 2. 启动采集端（每台被监控机）

双击 `run_agent.bat`，或命令行：

```bash
python agent.py --host 192.168.1.100 --port 5000 --token mySecret
```

参数：
- `--host` 主控端 IP（必填，填主控机局域网 IP）
- `--port` 端口（与主控端一致）
- `--token` 令牌（与主控端一致）
- `--fps` 初始帧率（默认 8）
- `--quality` JPEG 质量 10-95（默认 55，越低越省带宽）
- `--max-width` 缩放最大宽度，0=不缩放（默认 1280）
- `--monitor` 显示器编号，1=主屏（默认 1）

### 3. 操作

- **双击瓦片**：放大/还原单台画面
- **ESC**：退出放大
- **右键瓦片**：设置帧率 / 设置画质 / 断开此终端
- 瓦片左上角实时显示主机名和帧率，信号中断 3 秒显示"信号中断"

## 带宽估算

单台 1080p、质量 55、宽度缩放至 1280、8fps，实测约 **40 KB/帧 ≈ 320 KB/s**。10 台终端约 3 MB/s，百兆局域网轻松承载。降低 `--quality` 或 `--fps` 可进一步省带宽。

## 文件说明

| 文件 | 作用 |
|------|------|
| `protocol.py` | 通信协议（握手、帧收发、令牌校验） |
| `agent.py` | 采集端：屏幕抓取 + 推流 |
| `hub.py` | 主控端：TCP 服务 + PyQt5 网格界面 |
| `requirements.txt` | 依赖清单 |
| `run_hub.bat` / `run_agent.bat` | Windows 启动脚本 |

## 常见问题

**Q: 防火墙拦截？**
A: 在主控端放行对应端口（默认 5000）入站规则；采集端为出站连接，一般无需配置。

**Q: 多显示器只想要副屏？**
A: `agent.py --monitor 2`（编号对应 mss 的 monitors 顺序，1=全部虚拟桌面，2=主屏，3+=副屏）。

**Q: 想要远程控制（键鼠）？**
A: 本版刻意不含远程控制以避免高危行为。如确需，建议另行评估并独立实现，且务必做好授权与审计。
